# 测试质量宪法 v1.0

> TDD 就绪 + TDD 质量 + 测试可维护性
> 覆盖阶段：tickets → /tdd → /implement → /code-review

## 一、TDD 就绪门禁

> 触发：/to-tickets 完成后，/tdd 启动前
> 详细清单：`~/.claude/gate-checklists/tdd-readiness-checklist.md`

/tdd 启动前必须逐条确认 6 项：

| # | 规则 | 不通过的处理 |
|:-:|------|------|
| R1 | 错误码已定义 — 每个 API 端点的错误响应含 status 码 + body 格式 | 回 spec 补充 §Implementation Decisions |
| R2 | 测试数据已指定 — 前置准备中为具体值（ID/名称/参数），非占位符 | 回 ticket 补充 §前置准备 |
| R3 | API 入口已明确 — 探针目标为完整 HTTP URL，非文件路径 | 回 ticket 补充 §代码目录 |
| R4 | 权限机制已说明 — 后端如何读取/校验权限已写清 | 回 spec 补充 |
| R5 | 测试场景全部可执行 — 无"当前环境不可实现"的项 | 降级方案或标 `[human-verify]` |
| R6 | 测试环境已就绪 — Docker/DB/用户/Token 已验证可用 | 修复环境后重试 |

**不通过 → 立即停止，补齐再继续。** 不等、不猜、不假设"后面会补"。

## 二、TDD 执行规则

> 触发：/tdd 执行中
> 原理：AI agent 可以声称"完成"但没写好——测试是唯一不可伪造的证据。

| # | 规则 | 反例 |
|:-:|------|------|
| T1 | **测试先于实现** — 测试 commit 必须在实现 commit 之前 | "我先写完代码再补测试" |
| T2 | **逐点提交** — 每个 `[auto]` AC 完成 RED→GREEN 后立即 commit | 攒 5 个 AC 一次提交 |
| T3 | **不跳过** — 简单改动可跳过 spec 评审，但不可跳过 TDD RED 阶段 | "这个太简单了不用测试" |
| T4 | **Stub 必须明确** — 未实现的接口返回 NotImplementedError / 501 / `throw new Error('Not implemented')` | 返回 `null` / `[]` / 404 冒充 🔴 |
| T5 | **一个 ticket 一个 RED commit** — 全部 AC 的失败测试写完后统一提交，标记 `TDD: RED — ticket NNN` | 部分测试仍在暂存区就开始实现 |

> 详细门禁：`~/.claude/gate-checklists/test-checklist.md`（T1-T6，含签出清单）

## 三、测试接缝

> 触发：/to-spec 撰写 Testing Decisions 时
> 模板：`issues/test-plan-template.md`

| 层级 | 定义 | 适用 |
|------|------|------|
| E2E / API 探针 | 从外网/curl/Playwright 打真实 API | 用户可见行为、跨模块集成、回归 |
| 模块接口 | 调 public API/函数，不穿透实现 | 逻辑正确性、接口契约 |
| 单元 | 纯函数/算法/状态机 | 复杂计算、parser、状态迁移 |

**优先最高可用接缝**：能从 API 层测就不调内部函数；能从 public 方法测就不测 private。

## 四、测试可维护性

| # | 规则 |
|:-:|------|
| M1 | 测试测试行为，不测实现 — 重构内部逻辑不应破坏测试 |
| M2 | 一个测试文件对应一个 ticket，文件名含 ticket ID（`test_ticket_001.py`） |
| M3 | 测试数据独立（fixture/factory），不依赖生产数据 |
| M4 | `[human-verify]` AC 在测试文件中用 TODO 标注，不遗漏 |
